# WRAITH V7 BUG REPORT — Sevro Engine Cannot Fill React/Greenhouse Forms

**Date:** 2026-03-20
**Version:** Wraith v6 (post-fix, claimed all v6 bugs resolved)
**Engine:** Sevro (Phase 3) + QuickJS

---

## Test Target

- **URL:** `https://job-boards.greenhouse.io/agencywithin/jobs/5056863007`
- **Job:** AI Engineer at WITHIN
- **Platform:** Greenhouse (React-based job application forms)

---

## What Was Tested

1. `browse_navigate` — loaded the page, returned full DOM snapshot with all form fields and @ref IDs. **WORKS.**
2. `browse_fill` — called on 25+ fields (First Name, Last Name, Email, Phone, custom questions, EEO). Every call returned `"FILLED (native_events): <value>"`. **APPEARS TO WORK BUT DOESN'T.**
3. `browse_upload_file` — uploaded resume .docx, returned `"OK: uploaded matt_gates_resume_ai.docx (11403 bytes)"`. **STATUS UNKNOWN** (can't verify).
4. `browse_snapshot` — taken after all fills. **ALL FIELDS SHOW EMPTY `""`.**
5. `browse_click` on Submit button — returned `"CLICKED: Submit application"`. **PAGE DID NOT CHANGE.** Still on same form, same URL, same snapshot.
6. `browse_eval_js` — attempted React nativeInputValueSetter trick to force value binding. **JS returned success string** but snapshot still empty.
7. `browse_eval_js` — attempted to read back input values via querySelectorAll. **FAILED:** `"JavaScript evaluation failed: JS execution failed: Exception generated by QuickJS"`
8. `browse_screenshot` — **"Not available in Sevro (Phase 3)"**

---

## Root Cause Analysis

### The fills are theater

`browse_fill` with `native_events` strategy reports success, but Sevro is not a real browser. It cannot:

- Run React's synthetic event system
- Trigger React's `onChange` handlers on controlled components
- Dispatch events through a real browser event loop
- Execute real DOM APIs (HTMLInputElement.prototype doesn't exist in QuickJS)

Greenhouse uses React controlled inputs. The only way to fill them is:
1. A real Chromium/CDP session that dispatches keyboard events through the real event loop
2. Or React's internal fiber/props manipulation (requires a real JS engine with React loaded)

### QuickJS is not a browser

`browse_eval_js` runs in QuickJS, which is a standalone JS engine — not a browser environment. It doesn't have:
- `document.querySelectorAll` (or it's a stub that doesn't reflect real DOM state)
- `window.HTMLInputElement.prototype`
- React runtime
- Real event dispatching

The first eval_js call "succeeded" because it likely ran against a simulated DOM that accepted property sets. The second call with `.forEach` crashed QuickJS entirely.

## Specific API Issues

### 1. `browse_fill` returns false success
- Returns `"FILLED (native_events): Matt"` when the value did NOT persist
- Should return an error or warning when the engine cannot verify the fill stuck
- At minimum, the tool should read back the input value after setting it and confirm it matches

### 2. `browse_snapshot` doesn't reflect filled values
- After 25+ successful fill calls, snapshot shows all fields as `""`
- Either the fills didn't work (most likely) or snapshot doesn't read `.value` properties

### 3. `browse_eval_js` is unreliable
- QuickJS crashes on standard DOM APIs like `querySelectorAll` + `.forEach`
- No error detail — just "Exception generated by QuickJS"
- Users cannot debug or work around fill issues

### 4. `browse_screenshot` unavailable
- "Not available in Sevro (Phase 3)"
- No way to visually verify what the page looks like
- Critical for debugging form fill issues

### 5. `browse_click` on submit returns success with no state change
- Submit button click returned success but page didn't navigate or show confirmation
- Should detect if the page state changed after click and report accordingly

---

## Requested Fixes

### Critical (blocks all form automation)
1. **Add Chromium/CDP backend** for form-filling on JS-heavy sites. Sevro is great for navigation/extraction/scraping but cannot interact with React forms.
2. **Or fix `browse_fill` to dispatch real keyboard events** character-by-character through a real event loop, not just set `.value`.

### High Priority
3. **`browse_fill` should verify fills** — read back the value after setting and return error if it doesn't match.
4. **`browse_snapshot` should show actual input values** — read `.value` property, not just default attribute.
5. **`browse_eval_js` should use a real JS engine** or clearly document QuickJS limitations.

### Nice to Have
6. **`browse_screenshot` support in Sevro** or clear docs that it's not available.
7. **`browse_click` should report page state changes** (URL change, new elements, error messages).

---

## Workaround

None available within Wraith. Must use Playwright MCP or Chrome MCP (claude-in-chrome) for React-based job application forms.
